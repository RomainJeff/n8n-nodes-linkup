# Trigger Nodes — Poll, Webhook, and Event Patterns

Trigger nodes start workflows and supply initial data. They use programmatic style only (declarative does not support triggers).

---

## Trigger Node Types

| Type | Interface | How It Works | Example |
|------|-----------|-------------|---------|
| **Poll** | `IExecuteFunctions` | Runs on schedule, checks for new data | Check RSS feed, new emails |
| **Webhook** | `IWebhookFunctions` | Receives HTTP requests at a unique URL | GitHub webhooks, Stripe events |
| **Event** | `ITriggerFunctions` | Listens for events (SSE, WebSocket, etc.) | Real-time chat, live updates |

---

## Poll Trigger

Runs on a schedule and checks for new data since the last poll.

### Key Differences from Action Nodes

- `group: ['trigger']` instead of `['transform']`
- `inputs: []` — no inputs (it starts the workflow)
- `polling: true` — enables the scheduling UI
- Uses `getWorkflowStaticData('node')` for state between polls

### Complete Poll Trigger Example

```typescript
import {
  NodeConnectionTypes,
  type IExecuteFunctions,
  type INodeExecutionData,
  type INodeType,
  type INodeTypeDescription,
} from 'n8n-workflow';

export class MyServiceTrigger implements INodeType {
  description: INodeTypeDescription = {
    displayName: 'MyService Trigger',           // MUST suffix with "Trigger"
    name: 'myServiceTrigger',                   // MUST suffix with "Trigger"
    icon: 'file:myservice.svg',
    group: ['trigger'],                         // 'trigger' group
    version: 1,
    subtitle: '={{$parameter["event"]}}',
    description: 'Starts the workflow when MyService events occur',
    defaults: { name: 'MyService Trigger' },
    inputs: [],                                 // NO inputs for triggers
    outputs: [NodeConnectionTypes.Main],
    polling: true,                              // enables schedule UI
    credentials: [
      {
        name: 'myServiceApi',
        required: true,
      },
    ],
    properties: [
      {
        displayName: 'Event',
        name: 'event',
        type: 'options',
        noDataExpression: true,
        options: [
          {
            name: 'New Contact',
            value: 'newContact',
            description: 'Triggers when a new contact is created',
          },
          {
            name: 'Updated Contact',
            value: 'updatedContact',
            description: 'Triggers when a contact is updated',
          },
        ],
        default: 'newContact',
      },
    ],
  };

  async execute(this: IExecuteFunctions): Promise<INodeExecutionData[][]> {
    // Static data persists between polls
    const staticData = this.getWorkflowStaticData('node');
    const event = this.getNodeParameter('event', 0) as string;

    // Get the last checked timestamp (or default to 1 hour ago)
    const lastChecked = staticData.lastChecked as string
      || new Date(Date.now() - 3600000).toISOString();

    let url = 'https://api.myservice.com/v1/contacts';
    const qs: Record<string, string> = { since: lastChecked };

    if (event === 'updatedContact') {
      qs.filter = 'updated';
    }

    const response = await this.helpers.httpRequestWithAuthentication.call(
      this,
      'myServiceApi',
      { method: 'GET', url, qs, json: true },
    );

    const items = (response as { data: Array<Record<string, unknown>> }).data;

    // Update the last checked timestamp for next poll
    staticData.lastChecked = new Date().toISOString();

    if (!items || items.length === 0) {
      return [[]];    // No new data — workflow doesn't execute
    }

    return [this.helpers.returnJsonArray(items)];
  }
}
```

### Poll Trigger Key Points

- **Static data** (`getWorkflowStaticData('node')`) persists between polls — use it to track "last seen" timestamps/IDs
- **Return empty array** `[[]]` when no new data — the workflow won't execute
- **Schedule is automatic** — n8n handles the cron/interval, you just check for new data
- The schedule interval is configured by the user in the n8n UI

## Webhook Trigger

Receives HTTP requests at a unique URL generated by n8n.

### Key Differences from Action Nodes

- `group: ['trigger']`
- `inputs: []`
- `webhooks` array defines webhook endpoints
- Implements `webhook()` method instead of `execute()`
- Uses `IWebhookFunctions` interface

### Complete Webhook Trigger Example

```typescript
import {
  NodeConnectionTypes,
  type IWebhookFunctions,
  type IWebhookResponseData,
  type INodeType,
  type INodeTypeDescription,
} from 'n8n-workflow';

export class MyServiceWebhookTrigger implements INodeType {
  description: INodeTypeDescription = {
    displayName: 'MyService Webhook Trigger',
    name: 'myServiceWebhookTrigger',
    icon: 'file:myservice.svg',
    group: ['trigger'],
    version: 1,
    description: 'Starts the workflow when MyService sends a webhook',
    defaults: { name: 'MyService Webhook Trigger' },
    inputs: [],
    outputs: [NodeConnectionTypes.Main],
    webhooks: [
      {
        name: 'default',
        httpMethod: 'POST',
        responseMode: 'onReceived',     // respond immediately
        path: 'webhook',
      },
    ],
    properties: [
      {
        displayName: 'Event Type',
        name: 'eventType',
        type: 'options',
        noDataExpression: true,
        options: [
          { name: 'All Events', value: '*' },
          { name: 'Contact Created', value: 'contact.created' },
          { name: 'Contact Updated', value: 'contact.updated' },
          { name: 'Contact Deleted', value: 'contact.deleted' },
        ],
        default: '*',
      },
    ],
  };

  async webhook(this: IWebhookFunctions): Promise<IWebhookResponseData> {
    const body = this.getBodyData();
    const headerData = this.getHeaderData();
    const eventType = this.getNodeParameter('eventType') as string;

    // Optional: verify webhook signature
    const signature = headerData['x-webhook-signature'] as string;
    if (signature) {
      // Verify signature logic...
    }

    // Filter by event type
    const receivedEvent = (body as Record<string, unknown>).event as string;
    if (eventType !== '*' && receivedEvent !== eventType) {
      // Event doesn't match filter — ignore
      return {
        noWebhookResponse: true,    // don't trigger workflow
      };
    }

    return {
      workflowData: [
        this.helpers.returnJsonArray(body as Record<string, unknown>),
      ],
    };
  }
}
```

### Webhook Methods (IWebhookFunctions)

| Method | Returns | Description |
|--------|---------|-------------|
| `this.getBodyData()` | `IDataObject` | Request body |
| `this.getHeaderData()` | `object` | Request headers |
| `this.getQueryData()` | `IDataObject` | Query string params |
| `this.getRequestObject()` | `Request` | Full Express request |
| `this.getResponseObject()` | `Response` | Express response (for custom responses) |
| `this.getParamsData()` | `object` | URL path params |
| `this.getNodeParameter('name')` | `any` | Node parameter value |

### Webhook Response Modes

| Mode | Behavior |
|------|----------|
| `'onReceived'` | Respond immediately (200 OK), then execute workflow |
| `'lastNode'` | Wait for workflow to complete, respond with last node's output |
| `'responseNode'` | Wait for a specific "Respond to Webhook" node |

### Webhook with Custom Response

```typescript
async webhook(this: IWebhookFunctions): Promise<IWebhookResponseData> {
  const res = this.getResponseObject();

  // Custom response
  res.status(200).json({ received: true, timestamp: new Date().toISOString() });

  return {
    noWebhookResponse: true,    // we already responded
    workflowData: [
      this.helpers.returnJsonArray(this.getBodyData() as Record<string, unknown>),
    ],
  };
}
```

### Webhook Registration (for APIs that require subscription)

Some APIs require you to register a webhook URL. Override `webhookMethods`:

```typescript
webhookMethods = {
  default: {
    async checkExists(this: IHookFunctions): Promise<boolean> {
      // Check if webhook is already registered
      const staticData = this.getWorkflowStaticData('node');
      const webhookId = staticData.webhookId as string;
      if (!webhookId) return false;

      try {
        await this.helpers.httpRequestWithAuthentication.call(this, 'myServiceApi', {
          method: 'GET',
          url: `https://api.myservice.com/webhooks/${webhookId}`,
          json: true,
        });
        return true;
      } catch {
        return false;
      }
    },

    async create(this: IHookFunctions): Promise<boolean> {
      const webhookUrl = this.getNodeWebhookUrl('default');
      const staticData = this.getWorkflowStaticData('node');

      const response = await this.helpers.httpRequestWithAuthentication.call(this, 'myServiceApi', {
        method: 'POST',
        url: 'https://api.myservice.com/webhooks',
        body: {
          url: webhookUrl,
          events: ['*'],
        },
        json: true,
      });

      staticData.webhookId = (response as Record<string, unknown>).id;
      return true;
    },

    async delete(this: IHookFunctions): Promise<boolean> {
      const staticData = this.getWorkflowStaticData('node');
      const webhookId = staticData.webhookId as string;
      if (!webhookId) return true;

      try {
        await this.helpers.httpRequestWithAuthentication.call(this, 'myServiceApi', {
          method: 'DELETE',
          url: `https://api.myservice.com/webhooks/${webhookId}`,
          json: true,
        });
      } catch {
        // Ignore errors on delete
      }

      delete staticData.webhookId;
      return true;
    },
  },
};
```

## Event Trigger (ITriggerFunctions)

For real-time event streams (Server-Sent Events, WebSockets, etc.).

### Key Differences

- Implements `trigger()` method
- Uses `ITriggerFunctions` interface
- Must handle cleanup via `closeFunction`

### Event Trigger Example

```typescript
import {
  NodeConnectionTypes,
  type ITriggerFunctions,
  type ITriggerResponse,
  type INodeType,
  type INodeTypeDescription,
} from 'n8n-workflow';

export class MyServiceEventTrigger implements INodeType {
  description: INodeTypeDescription = {
    displayName: 'MyService Event Trigger',
    name: 'myServiceEventTrigger',
    icon: 'file:myservice.svg',
    group: ['trigger'],
    version: 1,
    description: 'Listens for real-time events from MyService',
    defaults: { name: 'MyService Event Trigger' },
    inputs: [],
    outputs: [NodeConnectionTypes.Main],
    credentials: [
      { name: 'myServiceApi', required: true },
    ],
    properties: [
      {
        displayName: 'Channel',
        name: 'channel',
        type: 'string',
        default: '',
        required: true,
        description: 'The event channel to listen to',
      },
    ],
  };

  async trigger(this: ITriggerFunctions): Promise<ITriggerResponse> {
    const channel = this.getNodeParameter('channel') as string;
    const credentials = await this.getCredentials('myServiceApi');

    // Example: connect to SSE stream
    const EventSource = require('eventsource');
    const es = new EventSource(
      `https://stream.myservice.com/events/${channel}`,
      { headers: { Authorization: `Bearer ${credentials.apiKey}` } },
    );

    es.onmessage = (event: MessageEvent) => {
      const data = JSON.parse(event.data);
      // Emit event to n8n workflow
      this.emit([this.helpers.returnJsonArray([data])]);
    };

    es.onerror = (error: Event) => {
      this.emitError(new Error('SSE connection error'));
    };

    // Cleanup function — called when workflow is deactivated
    const closeFunction = async () => {
      es.close();
    };

    return { closeFunction };
  }
}
```

### ITriggerFunctions Methods

| Method | Description |
|--------|-------------|
| `this.emit(data)` | Send data to the workflow (triggers execution) |
| `this.emitError(error)` | Report an error |
| `this.getNodeParameter('name')` | Get parameter value |
| `this.getCredentials('credName')` | Get credentials |
| `this.getWorkflowStaticData('node')` | Persistent state |
| `this.helpers.httpRequest(options)` | HTTP requests |

## Summary

| Trigger Type | Best For | Method | Frequency |
|-------------|----------|--------|-----------|
| **Poll** | APIs without webhooks, checking for changes | `execute()` | On schedule (user-configured) |
| **Webhook** | APIs with webhook support, real-time events | `webhook()` | On demand (push) |
| **Event** | SSE, WebSocket, long-lived connections | `trigger()` | Real-time (stream) |

**Choose based on what the external service supports:**
1. If it supports webhooks → Webhook trigger
2. If it has real-time streams → Event trigger
3. Otherwise → Poll trigger
